# Batch 79 competition notes

This batch focuses on rendered usability rather than new content.

## Why this matters

Even when competitor content is weaker, some pages still feel easier to use on mobile because:

- results appear exactly where the user expects after tapping calculate
- anchored help links do not disappear behind sticky headers
- button rows and quick-action links stack more cleanly on narrow screens
- FAQ controls are easier to tap and scan

## Gap closed here

WorkRightsUK already had stronger content and route logic on many core pages. This batch narrows the remaining mobile/rendered-state gap by making the shared interaction layer calmer and more reliable.

# Batch 64 - Competition notes

## Competitor pattern being matched
The stronger competitor homepages reduce decision time. They do not just describe the site; they tell the user what to click first.

## Gap being closed
WorkRightsUK already had strong core routes, but the homepage still spent too much space describing the site before pushing the user into the right first page.

## What this batch does better
- Moves the compare route higher for users who are unsure.
- Surfaces final pay earlier as the bridge route for mixed payslip/package problems.
- Makes the homepage links read like user questions rather than directory labels.
- Adds a simple decision order so users judge the immediate problem before the whole package headline.

## Why this matters
This should reduce hesitation on first landing and make the homepage feel more like a decision tool rather than a project hub.
